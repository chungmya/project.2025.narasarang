AD009 React Tools
Transform: HTML to JSX [localhost 설치] 
							🟢 설치 및 실행 방법은 다음과 같습니다.
							1. 터미널을 열고 프로젝트 폴더로 이동합니다.
							2. 다음 명령어를 순서대로 실행하세요:
							- yarn
							- yarn dev (개발용 실행)
							- 또는, 배포용으로 실행하려면 yarn build && yarn start
							- 추가적으로 yarn이 설치되어 있지 않다면, 먼저 Node.js와 yarn을 설치해야 합니다.

							🟢 Yarn 설치
							- npm이 설치되어 있다면: npm install --global yarn
							- yarn --version

							🟢 버전을 전환: 현재 설치된 Node.js 버전(18.18.0)이 프로젝트에서 요구하는 버전(16.x)과 맞지 않을때
							- nvm install 16
							- nvm use 16
							- node -v

							🟢 Windows에서 nvm(Node Version Manager)을 설치
							1. nvm-windows 공식 GitHub에서 최신 nvm-setup.zip 파일을 다운로드합니다.
							2. 압축을 풀고 nvm-setup.exe 파일을 실행
							- nvm version
							4. nvm으로 Node.js를 설치한 후, 터미널을 완전히 종료했다가 다시 열어야 환경변수가 적용
							5. 이제 원하는 Node.js 버전을 설치하고 사용할 수 있습니다
							- nvm install 16
							- nvm use 16
							- node -v

							🟢 nvm으로 node 최신 버전으로 변경
							1. 최신 LTS 버전 설치
							- nvm install lts
							2. 최신 LTS 버전 사용
							- nvm use lts
							3. 또는, 최신 버전 전체를 설치/사용하려면
							- nvm install latest
							- nvm use latest
							4. 현재 적용된 node 버전 확인
							- node -v

							🟢 프로젝트마다 다른 Node.js 버전을 사용하려면, 각 프로젝트 폴더에
							.nvmrc 파일을 만들어 원하는 버전을 지정하면 됩니다.
							1. 프로젝트 루트에 .nvmrc 파일 생성
							2. 파일 안에 사용할 Node.js 버전(예: 16.20.0 또는 16)을 입력
							3. 해당 프로젝트 폴더에서 아래 명령어 실행
							그러면 .nvmrc에 지정된 버전으로 자동 전환됩니다.
							- nvm use

							Tip:
							여러 프로젝트에서 각각 다른 Node.js 버전을 쉽게 관리할 수 있습니다.
							.nvmrc 파일만 프로젝트마다 다르게 작성하면 됩니다.

							모달 내 상호작용을 최소화한다.
							모달은 사용자의 행동을 통해 실행한다
							- 다만, 사용자에게 인증 세션 만료를 안내하거나 서비스의 메인 화면에서 사용자에게 긴급한 정보를 안내해야 하는 경우에는 자동으로 실행되는 모달을 사용할 수 있다.
							모달과 내부 요소의 초점 이동 순서를 논리적으로 제공한다.
							- 열기 버튼을 실행하여 창이 활성화되었을 때, 키보드 초점은 모달 자체 또는 모달 내에서 상호작용이 가능한 첫 번째 요소로 이동해야 하며, 모달을 닫거나 취소했을 때 초점은 모달 열기 버튼으로 이동해야 한다.
							모달의 닫기 버튼은 모달의 가장 마지막 요소로 마크업한다
06. 배지(Badge)

							배지의 배경색으로 주조색을 사용하지 않는다
							배지는 접근성과 명확한 이해를 위해 항상 텍스트 레이블을 제공해야 한다.
07. 아코디언(Accordion) 
							사용금지: 하위 또는 상위 요소로 이미 아코디언 컴포넌트가 사용되었거나 탭 패턴이 사용된 경우

							- button을 사용하거나 role="button"을 사용
							패널 영역에 스크롤이 발생하지 않도록 한다.
							아코디언이 접혀 있는 경우 사용자가 확장 가능한 영역이 있음을 알 수 있도록 'v' 또는 '+' 아이콘이, 펼쳐진 경우에는 ' v ', '-' 아이콘이 제공되어야 한다.
							아코디언 헤더 내부의 제목은 화면의 다른 콘텐츠의 구조에 맞추어 h1~h6 태그로 마크업한다
							아코디언의 확장 상태에 대한 정보를 텍스트 정보로 제공(펼치기/접기, 축소됨/확장됨)

							-- 이미지와 인접한 텍스트 설명과 동일한 설명
							-- 이미지에 없는 추가적인 정보
							이미지 자체는 대화형 콘텐츠가 아니므로 키보드 초점이 접근하지 않아야 하며, 마우스 클릭에 반응하지 않아야 한다
09. 캐러셀(Carousel) 
							자동 회전: 자동 회전을 정지할 수 있는 컨트롤을 제공
							- 자동 재생 캐러셀에서 정지 버튼이 상호작용 가능한 첫 요소로 제공되어야 한다.
							슬라이드 수를 5개 이내로 사용한다.
							캐러셀 항목의 텍스트 콘텐츠는 1~2줄로 작성한다.
							캐러셀 항목의 콘텐츠의 윤곽이 명확하게 드러나도록 표현한다.
							캐러셀 관련 컨트롤, 항목 탐색 식별자가 캐러셀 항목과 중첩되지 않도록 한다.
							캐러셀 관련 컨트롤과 항목 탐색 식별자를 일관된 위치에 표시한다
							전체 슬라이드 수와 사용자의 탐색 위치를 표시한다
							표시할 캐러셀 항목 수가 1개인 경우 캐러셀과 관련된 단서를 숨긴다
							캐러셀 관련 컨트롤 요소를 쉽게 조작할 수 있도록 충분히 크게 표현한다.
							- 마우스 상호작용에 대해서는 17px × 17px, 터치 상호작용에 대해서는 44px × 44px
							드래그 앤 드롭/스와이프로 슬라이드를 전환할 수 있도록 한다.
10. 탭(Tab)

							- 탭과 탭 패널의 배경색을 동일하게 지정
							탭에 사용 불가 상태를 사용하지 않는다.
							탭의 항목 수가 많아 일렬로 배치하기 어려운 경우 별도의 탐색 컨트롤을 제공한다.
							- 2열 안됨
							탭의 선택 상태를 색상으로만 구분하지 않는다
							- 밑줄 제공, 1px 이상의 테두리 차이, 식별자 제공 등의 방법으로 크기나 형태 차원에서 정보를 구분
							- 탭 버튼에 role="tab"로 역할을 부여하고 선택된 탭에 aria-selected="true"로 상태 정보를 전달하거나 title="현재 탭", title="선택됨"으로 선택 상태에 대한 정보를 제공
							role="tablist", 탭 버튼은 role="tab", 탭 패널은 role="tabpanel"로 역할을 부여한다.
							- 탭 버튼에는 aria-controls="[상응하는 탭 패널 id 속성값]", aria-labelledby="[상응하는 탭 버튼 id 속성값]"을 설정하여 탭과 패널 간 연결
11. 표(Table) 
							1. 열 구분선의 사용을 지양한다.
							2. 양적 데이터를 전달하기 위한 숫자는 오른쪽으로 정렬한다
							3. 텍스트 데이터, 명목 데이터는 왼쪽으로 정렬한다
